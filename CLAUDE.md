# CLAUDE.md - Boxing App 프로젝트 규칙

## 1. 팀 작업 규칙

- 1인 개발 + Claude 협업 체제
- 작업 규칙은 프로젝트 진행하며 지속 추가
- **중요 규칙**
  **파일을 바로 수정하지 않는다** — 요청을 받으면 먼저 현재 상태를 파악한다.
  **현재 코드, 구조, 문제점을 먼저 파악한다** — 수정 전에 반드시 관련 코드를 읽고 이해한다.
  **계획만 먼저 출력한다** — 사용자 확인을 받은 후에 코드를 수정한다.
- 기본적으로 객체 지향(OOP) 스타일을 적용
- 중복 코드가 없이 최대한 효율적으로 작성
- 모든 해상도에 대응할 수 있도록 반응형 디자인 적용
- 기능 확장이 유연하도록 설계
- **작업 로그 기록**: 파일을 수정/생성할 때마다 `working_log/YYYYMMDD.md`에 작업 내용을 간단히 정리하여 추가한다.
  - 파일이 있으면 이어서 작성, 없으면 새로 생성
  - 당일 날짜 기준으로 파일명 결정

## 2. 프로젝트 목표와 방향

- **앱 이름**: Boxing App (가칭)
- **프레임워크**: Flutter (Dart)
- **대상 플랫폼**: Android, iOS
- **핵심 기능 (1차)**:
  - 운동 기록 (세트, 라운드, 종목 등)
  - 트레이닝 타이머 (라운드 타이머, 휴식 타이머)
- **향후 확장**: 1차 기능 완성 후 기획
